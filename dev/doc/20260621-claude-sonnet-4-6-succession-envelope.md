# Eccentricaspects.com succession envelope

## What this is

`eccentricaspects.com` is a memorial archive of the personal blog of
**Nick Wormley** ("cynick"), a railroad historian, photographer, and
software engineer from Lowertown, Saint Paul. The site is a frozen
archive — no new content will ever be created, though additional
recovered artifacts (photographs) may be added if found.

This document contains everything a successor needs to keep the site
running, or to recover access if the current maintainer becomes
unavailable.

---

## How the site works

The site is a set of static HTML/CSS/JS files served by GitHub Pages.
There is no database, no build step, no server-side code, no container,
and no CI/CD pipeline. It was originally generated by Hakyll (a Haskell
static site generator) and has been restored from archived copies.

| Component | Location |
|-----------|----------|
| Site content | `github.com/eccentricaspects/eccentricaspects` (public repo) |
| Hosting | GitHub Pages (free, automatic HTTPS via Let's Encrypt) |
| DNS | Cloudflare (free plan) |
| Domain registration | Cloudflare Registrar (transfer from Route53 pending/complete) |

## If something breaks

### Site is down but DNS resolves

GitHub Pages is having an outage. Check `githubstatus.com`. No action
required — it will recover automatically.

### DNS stops resolving

1. Log in to Cloudflare (`dash.cloudflare.com`) with `eccentricaspects@gmail.com`
2. Verify the DNS records above are present
3. Verify the zone status is "Active"

### Domain expired

1. Log in to Cloudflare
2. Go to Domain Registration → Manage
3. Renew the domain
4. If Cloudflare account is inaccessible, re-register `eccentricaspects.com`
   at any registrar and point DNS to GitHub Pages (records listed above)

### GitHub account is locked or inaccessible

1. Use `eccentricaspects@gmail.com` to reset the GitHub password
2. If the Gmail account is also inaccessible:
   - The repo is public — fork it under any GitHub account
   - Re-enable GitHub Pages on the fork
   - Update DNS to point to the new `<username>.github.io` CNAME

### Complete disaster recovery

The site can be reconstructed from scratch with any GitHub account and
any domain registrar. The repo is public and the site is fully
self-contained. The steps are:

1. Fork or clone `github.com/eccentricaspects/eccentricaspects`
2. Enable GitHub Pages on the fork
3. Register or transfer `eccentricaspects.com` (or any domain)
4. Add A/AAAA/CNAME records pointing to GitHub Pages
5. Update the `CNAME` file in the repo to match the new domain
